Quick Start

Overview

In this document we will show you how to replicate data from Oracle or MySQL database to MongoDB, using Tapdata Cloud.

Architecture

There are 4 components in this diagram:

  • Tapdata Management Portal: The main web application, you can log on to manage your jobs. This is managed by Tapdata team.
  • Tapdata Agent: You need to download this to install on your local server in your data center.
  • Source DB: Oracle, MySQL, SQLServer etc. The source database server you would like to read data from.
  • MongoDB: Desitination, could be local MongoDB server or in the public cloud.

Get Started to use Tapdata Cloud

We have recorded a quick video to showcase the Oracle - MongoDB replication. Please access the video here: https://www.youtube.com/watch?v=rWoEfi9TSDw&t=16s

Below are the steps with illustrations to walk you through the proceduce to replicate data from relational database to MongoDB.

1. Sign into Tapdata and download replication agent

  • Log onto Tapdata Cloud ( https://cloud.tapdata.io )

  • Go to "Agent Download" page and download the Tapdata agent software(Requires Linux operating system)

  • Uncompress the agent package

  • Update Access Code in application.yml file. You can get the Access Code from User Info page.

  • Start agent with ./agent.sh start command

  • Go to "Agents" page, you should see your agent listed there.

2. Create an MySQL(or any other supported database) source connection

  • Click Connections link in the left navi bar

  • Click "Create"

  • Select "MySQL" as Database Type

  • Enter a connection name

  • Select "SOURCE" as Connection Type

  • Enter the database host/port

  • Enter MySQL's database name(Often time it's database you want to use)

  • Enter Username/Password

  • Click Save, wait for the "Test" to complete

3. Create target connection

  • Click Connections link in the left navi bar

  • Click "Create"

  • Select "MongoDB" as Database Type

  • Enter a connection name

  • Select "Target" as Connection Type

  • Select URI or Form

  • Enter MongoDB Conenction URI or using the form to assist with the connection informaton

  • Click Save, wait for the "Test" to complete

More information about connections please refer to Manage Connections page

4: Create a new job

  • Go to Data Collect

  • Click "Create"

  • Enter Job Name

  • Select "Custom" for Sync Mode

  • Click "Source" icon and select the source connection just created

  • Click "Mapping", and refer to Edit Job Page for details about defining a relational to JSON mapping

  • Click "Target", select the target connection you created

  • Click "Save", then click "Start" on the job card. As the default setting, tapdata will do Initial Sync exclude CDC replication.

  • Click on the job card to go into the monitoring page for the job
